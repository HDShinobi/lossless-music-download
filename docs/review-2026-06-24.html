01Download from Sources
Senior Developer
BUG
Batch download calls queue.enqueue(track) with no service/quality — silently falls through to dlProviders.first
lib/screens/search_screen.dart:97–113
BUG
Quality options are 3 hardcoded radios — extension quality caps not read at all
lib/widgets/download_sheet.dart:18–22
GAP
No error classification — error_type is raw string, never mapped to enum. SpotiFLAC has DownloadErrorType driving distinct user messages
lib/providers/download_queue_provider.dart:212–217
Senior UI/UX
GOOD
Source filter pill is excellent — better than SpotiFLAC's approach. Keep as-is.
GAP
No confirmation of which source/quality was used after a successful download. User has no feedback that the right version was fetched.
GAP
Error messages don't explain what went wrong or offer next steps (retry, switch source, contact support).
02Download Process / Queue
Senior Developer
BUG
Status-mapping logic duplicated verbatim in two places
provider:312 · widget:252 — extract one shared utility
BUG
progressStream() uses .handleError((_){}) — deserialization errors silently swallowed, no fallback re-subscription
lib/services/backend_bridge.dart:92–95
GAP
Queue in-memory only — lost on restart, cross-session duplicate detection broken. Fix: persist completed entries to SharedPreferences.
CLEANUP
EventChannel busy-loop runs unconditionally even when nothing is downloading
android/MainActivity.kt:57–67
Senior UI/UX
BUG
Highest-impact UX issue: download button shows no state change after queuing. SpotiFLAC swaps to spinner → checkmark. Users can't tell if tap registered.
GOOD
Persistent _DownloadBar strip is excellent — better than SpotiFLAC's modal queue approach. Keep.
GAP
No pause / cancel per-track from the queue view. Users can dismiss but cannot re-order or retry inline.
03Library
Senior Developer
BUG
ALAC files from Tidal marked unverified — verified: ext == 'FLAC' misses ALAC
library_provider.dart:108 — add || ext == 'ALAC'
BUG
File.lengthSync() called synchronously for every entry. sizeBytes is not even rendered in the UI — remove or make async
lib/providers/library_provider.dart:100–102
GAP
Full re-scan on every library open. Backend supports incremental but Flutter always passes a fresh call. Migrate to NotifierProvider with incremental scans.
Senior UI/UX
GAP
No search or filter in library — the single largest daily-use gap. SpotiFLAC has search + sort + advanced filter. A growing collection is unusable without it.
BUG
Album grid tiles have no onTap. Tapping an album does nothing — breaks the core browsing expectation
lib/screens/library_screen.dart _AlbumTile
GOOD
Format/quality badges (FLAC/M4A/ALAC) and the verified checkmark are better information density than SpotiFLAC's library tiles.
04Metadata Embedding
Senior Developer
GAP
Embed flags (embedMetadata, embedMaxQualityCover, embedLyrics) hardcoded true — no user control. Add 3 toggles to settings backed by SharedPreferences.
GAP
Filename format hardcoded as {"{artist}/{album} ({year})/{track}. {title}"}. Make it user-editable with live preview in settings.
GAP
No ReplayGain support. SpotiFLAC-Module-Version has this capability in its Python backend.
Senior UI/UX
GAP
No metadata editor in library. SpotiFLAC has long-press → edit tags. Power users expect this for fixing incorrect metadata from the source.
GAP
No feedback after metadata write — user can't confirm tags were embedded correctly without opening a third-party player.
GOOD
ATMOS badge in track tile is a meaningful differentiator — not present in SpotiFLAC.
05Settings
Senior Developer
BUG
"Ask before download" toggle never read by download flow — sheet always appears regardless
settings_screen.dart:23–30 vs search_screen.dart:89–91
GAP
No download directory picker — path auto-resolved by AppDirs.downloadDir(), not user-configurable.
GAP
